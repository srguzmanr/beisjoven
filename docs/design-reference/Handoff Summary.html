TL;DR

The current homepage buries its own front page. Readers see 450px of chrome and a mute photo hero before any headline renders — on a site where 76.6% of traffic is mobile and only 8.9% of readers scroll past 50%. This redesign cuts header chrome in half, puts an editorial hero with a visible headline above the fold on every viewport, and introduces a category color system so the six editorial sections become navigable assets instead of a flat list.

We're shipping as a clickable prototype with three design bands (Safe · Default · Ambitious), two category-color modes, and a live-event toggle — so Product can make the brand-level calls before UX specs them and engineering builds them in Astro.

The problem we're solving

Visual audit · current homepage on 1440 desktop

The live site is a well-produced editorial product hiding behind a generic publisher chassis. Five specific failures compound:

  1. The hero is mute

    A full-bleed photo renders above the fold with no visible headline. The most important story of the day is represented by a basketball-celebration image — readers can't tell what it's about without scrolling. Museum poster, not news hero.

    Fix → headline overlay on mobile, prose block on desktop, category kicker on both.

  2. Header wastes ~450px before a headline

    Top bar (date + Contacto) → oversized logo + oversized search → flat nav → LIVE strip. Four stacked bands on a site where the fold is roughly 660px on mobile.

    Fix → single brand row, search behind icon, quick-hits ticker folded into nav band.

  3. Logo is small, search is oversized

    Visual weight is inverted — the search field outweighs the brand mark.

    Fix → BJ monogram + BEISJOVEN wordmark at ~28–36px; search collapses to a 44×44 icon.

  4. Nav is flat & underutilized

    GA4 shows 8 category clicks / 30 days. The nav isn't earning its space. Active state is a one-pixel red underline.

    Fix → 2px active underline in category accent color; mobile uses horizontal chip scroll with right-edge fade.

  5. Cards lack variety

    Hero and secondary cards share the same aspect, pill, and spacing. No signal that one story matters more than another.

    Fix → hero / standard / list / rail variants with specified ratios, type steps, and category left-border rules.

Our approach

Three bands instead of three competing proposals

Rather than ship three standalone designs, we packaged the decisions into one prototype with three toggleable design bands. Every other choice — category color, live event, mobile nav behavior — is orthogonal. This lets Product mix the header from one band with the hero from another without re-spec work.

Recommendation

Ship Default as the daily look. Keep Ambitious as a tentpole mode triggered manually for big events.

Default balances editorial seriousness with production sustainability. Ambitious is worth the investment occasionally but costs photo direction every day it runs. Safe is the A/B control we'd fall back to if Default fails a click-through guardrail post-launch.

Category color system — brand decision

Tradeoff laid out for a single call

Option A · Single gold accent

  • Calmer, more unified brand voice.
  • Zero risk of category clashing against photos.
  • Does less work for wayfinding — readers rely entirely on pill labels to distinguish sections.
  • Fits the "one paper, one voice" editorial philosophy.

Option B · Per-category accents (recommended)

  • MLB LIGAS MX SELECCIÓN SOFTBOL JUVENIL OPINIÓN
  • Makes Juvenil (our editorial moat) identifiable at a glance on any scroll position.
  • Section underline + 4px card left-border + pill all carry the accent — a single design rule applied three places.
  • Fully reversible if GA4 shows it's not moving category-click behavior.

Our read: ship with per-category accents (Option B). The colors are calibrated within the Typography Spec v1.1 low-chroma palette, so they read as editorial signage rather than a kids-book rainbow. The toggle stays in the codebase as useCatColor — one prop flip to revert.

Decisions we need from you

Please resolve before engineering picks up the ticket
  1. Decision

    Band choice for daily

    Safe · Default (our rec) · Ambitious. This determines the hero and header patterns engineering builds first.

  2. Decision

    Category color — on or off

    Per-category accents (our rec) vs single gold. Reversible post-launch via one prop, but drives card + pill + underline spec.

  3. Decision

    Quick-hits ticker — editorial ownership

    We designed it in. Who writes the 3–5 entries? Is it admin-editable in the CMS or scripted per-event?

  4. Risk

    Original photography pipeline

    Default & Ambitious heroes both depend on a daily lead photo with a clear focal subject. If the pipeline produces more event-scoreboard shots than editorial portraits, mobile gradient overlays will fight the image. UX should review 5–10 real candidate photos against the hero overlay before launch.

  5. Risk

    Category underutilization may be a content problem, not a design one

    GA4 shows 8 category clicks / 30d. Stronger nav design helps, but if category hubs themselves are underpopulated, the design won't move the needle alone. Worth parallel editorial review of each of the six hub pages.

  6. Scope

    Sponsor slot timing

    Phase 1 reserves the slot (970×90 desktop, 320×50 mobile) below the hero. UX should confirm Sprint 5 still owns the sponsor system design so we don't paint ourselves in.

Suggested next steps

Two weeks from review to tickets-ready

  1. Design review session PM + UX/UI

    30 min. Walk the prototype band-by-band on a 375 phone. Resolve the three Decisions above. This week.

  2. Photography audit UX + Editorial

    Pull 10 recent hero candidates from the photo library. Drop them into the Default hero on mobile. Confirm the gradient overlay holds. Flag any that don't.

  3. Engineering estimation Claude Code

    Break prototype into Astro components: Header, Hero, StoryCard, ListCard, CategoryRail, Gallery, Newsletter. Tokens are already token-ready in tokens.css.

  4. QA plan UX + Eng

    Visual regression on 375 / 768 / 1280. Lighthouse mobile target: LCP < 2.5s, CLS < 0.1. Above-fold must contain at minimum: logo, nav, one headline, one image.

  5. Success metrics PM

    Scroll-to-50% from 8.9% → target 18% at 30 days. Category clicks from 8/30d → target 60+/30d. Article completion rate hold ≥ 45%.
