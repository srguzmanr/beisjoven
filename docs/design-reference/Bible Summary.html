What's in Bible.html

Three parts · canonical · token-live
01

Product

Who we are, who we serve, how we file stories when pillars collide.

  • Vision — Somos el futuro del beis. Mission block, one line.
  • Six pillars with slugs (/liga-mexicana · /mlb · /softbol · /seleccion · /juvenil · /opinion) and display names (Ligas MX, MLB, Softbol, Selección, Juvenil, Opinión). Slugs are SEO-locked.
  • 7 classification rules, authoritative. Sport beats age (Softbol). MLB institution beats age. WBSC → Selección, WBC → MLB, BCL → Ligas. College baseball → Juvenil, college softball → Softbol.
  • Byline rule — Redacción Beisjoven always. Personal name never in Phase 1.
  • Audience snapshot — 76.6% mobile, 50/30 US/MX, 47% completion, 8.9% scroll-to-50%, primary viewports 375/412.
  • Phase — Phase 1 (content + audience). First sponsor closed Feb 2026.
02

Design System

Tokens from Spec v1.1 applied verbatim. No new tokens introduced.

  • Type scale — fluid clamp table: H1 (30→44) to Overline (11), rendered live.
  • Color — Brand (Navy, Red, Gold, White, Off-white, Rule, Muted) + 6 category accents.
  • Category pills — solid bg, white text (navy on Juvenil gold), Inter 700 · 11 · 0.08em · 4×10 · radius 2.
  • Cards — three variants: Standard 16:10, List 1:1, Hero 21:9. 4px category left border.
  • Spacing — gutter 16/32, section 48/64, card gap 16–24, prose 672, breakout 960, page 1200.
  • Link states — navy + 1px 40%-opacity underline → hover red → active darkened red → focus gold ring.
  • Pull quote — 4px category border, PJS 500 italic, 22/26, no decorative marks.
  • Accessibility floors — WCAG AA. Gold-on-white (2.67:1) called out as never body.
03

Page Architectures

Four templates. Module order authoritative. Components named; anatomy lives in Part 2.

  • Header (shared) — Brand · Nav · Quick Hits · conditional LIVE. Auto-hide mobile; shrink-on-sticky desktop. Icon-only search.
  • Homepage — Hero (Default baseline · HeroSplit first-class · HeroAmbitious unwired) · Sponsor reservation · Mixed feed + Más leídos · Juvenil rail · Gallery · 5 category sections · Newsletter · Footer.
  • Article — Breadcrumbs · Article Hero (Default / Overlay) · Header Block (kicker · H1 · deck · metadata) · Share Bar (sticky desktop / inline mobile) · Body (prose 672, breakouts 960) · Author Bio · Related 4-up · Newsletter · Footer.
  • Category — Category Header · Featured · 3-up article grid · Load More. Inherits Homepage card rules.
  • Author — Author Hero · stats · 3-up grid · Load More. Phase 1: single /autor/redaccion-beisjoven. Schema ready for Phase 2.

Structural notes

  • Token stack unchanged. tokens.css is shared across Bible, Spec, Prototype.
  • Sticky TOC with Part 1 / 2 / 3 anchors, scrollspy active state.
  • Token-live rendering — colors, type and pills render themselves, not described.
  • Viewports 375 + 1280, matching prototype.

Out of scope (held elsewhere)

  • Dark mode — Q3 2026 exclusion holds.
  • Admin panel — separate track.
  • Parts 4–6 — Locked decisions, stack, principles. PM owns in markdown companion.

Deferred to the markdown companion

Parts 4–6 · owned by PM · not rendered here

What the markdown doc carries

  • Part 4 — Locked decisions. Every resolved-and-binding call: byline rule, slug freeze, WBC→MLB exception, Juvenil reverse-text pill, sponsor-slot reservation pattern, HeroAmbitious-unwired posture. Dated, with owner.
  • Part 5 — Stack. Astro SSG + Tailwind config, deployment target, CMS shape (pillar enum + slug map), analytics (GA4), performance budgets (LCP < 2.5s, CLS < 0.1). Zero paid deps rule.
  • Part 6 — Editorial principles. Voice, headline heuristics, deck conventions, photo-direction floor, the one-paper-one-voice rule, the "paper of record for a beat that doesn't yet have one" posture. Process for classification edge cases not already covered by Rules 1–7.
Rationale: 4–6 are product doctrine — they change without visual consequence. Keeping them in markdown keeps Bible.html a pure visual/structural reference that can ship in any handoff without a thinking-out-loud essay attached.
