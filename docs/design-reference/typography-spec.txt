Executive Summary
This document defines the official typography scale and spacing system for Beisjoven’s Q3
2026 visual redesign. These specs govern every text element on the site and establish the
spatial rules that create visual rhythm across all pages.
Why this matters for Product: These tokens are the foundation every component builds
on. This spec unblocks ticket DESIGN-01 and all Sprint 4–5 implementation.
Why this matters for Marketing: Typography is the biggest factor in perceived editorial
quality. This scale positions Beisjoven alongside The Athletic and Baseball America. It
standardizes the visual language for branded content, sponsor decks, and cross-platform
assets.
Design Principles
Mobile-first. 76.6% of readers are on phones. Every size is defined at 375px first, then
scaled up fluidly.
Content is the hero. Typography and spacing make articles easier to read and
photography more impactful.
Readability over decoration. Generous sizes, comfortable line heights, ample
whitespace.
Benchmark-informed. Calibrated against The Athletic (premium reading) and Baseball
America (clean editorial).

Key Decisions
Body at 17px mobile / 18px desktop: The Athletic uses 18px desktop. Our 17px mobile
produces ~55–65 chars per line at 375px. Spanish runs ~15% longer than English, so the 1.7
line-height prevents cramping.
H1 at 30px mobile / 44px desktop: Baseball America headlines land at 28–32px mobile.
We sit at the top of that range to match our photography-forward aesthetic.
Negative tracking on headings: Plus Jakarta Sans is loose at display sizes. Tightening to
-0.02em on H1 creates editorial intentionality matching The Athletic.
Article body capped at 672px (42rem): Produces 65–75 chars per line at 18px — optimal
for sustained reading. Both The Athletic and Baseball America use similar column widths.
Pull Quote Visual Treatment
Left-border accent with italic text. No background color, no decorative quotation marks.
Clean and editorial, letting the quoted text carry the weight.

Why No Decorative Quote Marks
Large typographic quote marks require precise positioning across breakpoints and compete
with the category-colored border. The border-only treatment is cleaner, faster to implement,
and reinforces the category color system.

Design Rationale
Navy for default (not Red): Red links would collide with category pills and CTA buttons.
Navy provides clear contrast against Dark Gray body text while staying in the brand palette.
Always-visible underline: 76% mobile audience = no hover for discovery. Links must be
identifiable on first glance. The Athletic uses the same pattern.
No visited color: Distinct visited colors leak browsing history and add noise. Standard
practice for editorial sites.
Gold focus ring: Strong contrast against Navy text and white backgrounds. Meets WCAG
2.1 focus indicator requirements.
Responsive Strategy: CSS clamp()
Decision: CSS clamp() for fluid scaling. No third breakpoint. No abrupt swap at 768px.
How It Works
Each type size scales fluidly between its mobile minimum and desktop maximum. Values
interpolate linearly between 375px and 1280px viewport width. At any tablet width, readers
get a naturally proportional size.

Why clamp() over breakpoint swap: A jump at 768px is visible on iPad rotation. Fluid
scaling eliminates this. Claude Code writes one line per property instead of managing media
queries.
Browser support: 96%+ global support. No polyfill needed.
Spacing stays breakpoint-based: Only typography uses clamp(). Spacing tokens keep the
xl: breakpoint approach because spatial jumps are less perceptible than type jumps.
Article Body Architecture
The 672px constraint lives on a prose wrapper inside <article> — not on the article
element itself. This lets specific elements break out to wider widths.
HTML Structure
<article>
<div class="prose-wrapper max-w-[42rem] mx-auto px-4">
<!-- Headline, metadata, paragraphs, inline images -->
</div>
<div class="breakout max-w-[60rem] mx-auto px-4">
<!-- Gallery, video embed, data table -->
</div>
<div class="prose-wrapper max-w-[42rem] mx-auto px-4">

</div>
</article>

Dark Mode Scope
DARK MODE IS OUT OF SCOPE FOR Q3 2026.
All tokens in this spec are light-mode only. When dark mode is prioritized (tentatively Q4
2026), the type scale and spacing will not change — only colors get a dark layer via CSS
custom properties. The architecture is dark-mode-ready; no rework of this spec will be
needed.

Font Stack
Plus Jakarta Sans — All headings, pull quotes, display text. Google Fonts (500/600/700).
Free, open source.
Inter — All body, nav, buttons, metadata, captions. Google Fonts (400/500/600). Free, open
source.
Both fonts are already in production on beisjoven.com. No new loading required.
Implementation Notes
For Product / Claude Code: All values use CSS clamp() custom properties. Tailwind config
extends fontFamily and letterSpacing. A single CSS file + tailwind.config.mjs update = ticket
DESIGN-01.
For Marketing: For branded content and sponsor decks: Plus Jakarta Sans Bold for
headlines, Inter Regular for body. Minimum body text 16px on digital. Both fonts free on
Google Fonts for Canva, presentations, or print.
